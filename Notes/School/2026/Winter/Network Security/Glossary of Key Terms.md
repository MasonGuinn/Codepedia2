## Numerical & Special Characters

### 3DES

### 802.1x

## A

### Acceptable Use Policy (AUP)

- **Definition:** The structured document that outlines what is permitted to do and view when using company equipment and computers.
    

### Access Control List (ACL)

- **Definition:** Mechanism defining traffic or an event to apply an authorization control of allow or deny against. Often used interchangeably with the terms rule and filter in relation to firewalls.
    

### Access Control

- **Definition:** The process or mechanism of granting or denying use of a resource; typically applied to users or generic network traffic.
    

### Active Directory (AD)

- **Definition:** A directory service found in Windows operating systems used for managing permissions and access to networked resources.
    

### Active Hub

- **Definition:** A device that consolidates wiring so that a single cable can attach to a server and includes the capability to regenerate the signal to decrease latency.
    

### Active Stack

- **Definition:** The subset of the protocol stack that is presently in use (active) on the network.
    

### Active Threat

- **Definition:** A form of threat that takes initiative to seek out a target to compromise, such as hackers, intruders, or automated worms.
    

### Address Resolution Protocol (ARP)

- **Definition:** A non-authenticating broadcast query service that requests the Media Access Control (MAC) address from a system using a specific IP address.
    

### Addressing

- **Definition:** The assignment of a logical numbering system to the hosts on a network for the purposes of efficient traffic routing.
    

### Administrative Controls

- **Definition:** Procedural ways to limit a threat or risk, such as the application of permissions, use of security policies, and data classification.
    

### Advanced Persistent Threat (APT)

- **Definition:** A network attack in which an unauthorized person gains access to a network and stays there undetected for a long period to steal data.
    

### Adware

- **Definition:** Unwanted software that displays advertisements. Often linked with spyware.
    

### After-Action Alert

- **Definition:** A notification that occurs after an action was undertaken due to preset conditions being met.
    

### Agent

- **Definition:** A malicious software program (bot or zombie) distributed by a hacker to take over control of a victim‚Äôs computer.
    

### Aggregate / Aggregation

- **Definition:** To collectively consider or amass individual pieces as a whole. In logging, it refers to collecting disparate log information from various systems into one location.
    

### Alert Fatigue

- **Definition:** A condition leading to apathy or un-investigated dismissal of alarms due to a high volume of notifications.
    

### Algorithm

- **Definition:** A set of rules and procedures, usually mathematical, defining how processes like encryption and decryption operate.
    

### Allowlist

- **Definition:** A method of restricting access to only addresses, protocols, or other determiners deemed permissible.
    

### Alternate Data Stream (ADS)

- **Definition:** A feature of the NTFS file system that supports multiple resource forks for file objects; often used by hackers to hide files.
    

### Analytics

- **Definition:** The built-in ability to analyze and assess data patterns and conditions to identify characteristics and present results.
    

### Annualized Loss Expectancy (ALE)

- **Definition:** The calculation of the total loss potential across a year for a given asset and a specific threat.
    
- **Formula:** $ALE = SLE \times ARO$
    

### Annualized Rate of Occurrence (ARO)

- **Definition:** A probability prediction based on how many times in the next year a threat is likely to cause harm.
    

### Anomaly-Based Detection

- **Definition:** IDS/IPS detection based on a defined "normal." Traffic that fails to match the defined normal is considered an anomaly.
    

### Anonymity

- **Definition:** The ability for a network or system user to remain unknown.
    

### Anti-Forensics

- **Definition:** Behaviors, techniques, or tools used to hinder the forensic recovery of data or to prevent identification of an attack.
    

### Antivirus Scanner

- **Definition:** Software that identifies unique characteristics of malware (virus signatures) to detect and prevent infection.
    

### Appliance

- **Definition:** A hardware product dedicated to a single primary function (e.g., firewall, router) with a hardened OS.
    

### Application Layer (Layer 7)

- **Definition:** The top layer of the OSI model responsible for enabling communications with host software and the network protocol stack.
    

### Arbitrary Code Execution

- **Definition:** An exploit that allows a hacker to run any command line function on a compromised system.
    

### ARP Flooding / Spoofing

- **Flooding:** Overloading devices by sending many ARP requests.
    
- **Spoofing:** Falsifying ARP replies to trick the requestor into sending frames to the wrong destination.
    

### Artificial Intelligence (AI)

- **Definition:** The use of algorithms to mimic human behavior and intelligence.
    

### Asset Value (AV)

- **Definition:** The cumulative tangible and intangible value of an asset.
    

---

## B

### Backbone

- **Definition:** The main traffic pattern or core of an existing network.
    

### Backdoor

- **Definition:** Any access method or pathway that circumvents standard access or authentication mechanisms.
    

### Backup

- **Definition:** Making copies of data to protect against loss, allowing for restoration when necessary.
    

### Banner Grabbing

- **Definition:** Capturing or extracting banners from services to learn which service or version is active on a port.
    

### Baseline Establishment

- **Definition:** Creating a starting point for evaluating "normal" system or network traffic over time.
    

### Bastion Host

- **Definition:** A system designed and hardened specifically to serve as a frontline defense for a network.
    

### Behavior-Based Detection

- **Definition:** IDS/IPS detection based on a recording of real-world traffic used as a baseline for normal.
    

### Blind Spot

- **Definition:** An area of a network where there is no visibility into traffic or device activity.
    

### Blocklist

- **Definition:** Restricting access to any addresses or protocols explicitly listed as prohibited.
    

### Botnet

- **Definition:** A network of compromised systems (zombies/agents) controlled remotely by a "bot herder."
    

### Bridge

- **Definition:** A network device that forwards traffic between networks based on MAC addresses.
    

### Bring Your Own Device (BYOD)

- **Definition:** A policy allowing employees to connect personal devices to organization networks.
    

### Brute-Force Password Attack

- **Definition:** Attempting all possible valid combinations of characters to crack a password or encryption key.
    

### Buffer Overflow

- **Definition:** A condition where data exceeds a memory buffer's capacity and overflows into adjacent memory, often used to inject malicious code.
    

### Business Continuity Plan (BCP)

- **Definition:** A plan to maintain mission-critical functions during a problem that threatens business processes.
    

---

## C

### Caching

- **Definition:** Retention of internet content by a proxy server to speed up subsequent requests for the same data.
    

### Centralized Logging System

- **Definition:** Storing or copying log events from multiple devices to a single, central server (e.g., syslog).
    

### Certificate Authority (CA)

- **Definition:** A trusted third-party entity that issues digital certificates to verify identities.
    

### Challenge-Response

- **Definition:** A security process where a user's identity is tested by their response to a query (e.g., entering a code sent to a phone).
    

### Change Management

- **Definition:** A process for managing and tracking changes to system or network configurations to ensure stability.
    

### Chokepoint

- **Definition:** A controlled pathway through which all traffic must cross, allowing for deliberate filtering or monitoring.
    

### Ciphertext

- **Definition:** The encrypted, unusable output from a cryptographic function.
    

### Circuit Firewall

- **Definition:** A device that filters the initial creation of a session (state) but performs no subsequent filtering once established.
    

### Client/Server Network

- **Definition:** A network where specific computers (servers) host resources shared with other computers (clients).
    

### Collision Resistance

- **Definition:** A property of a hash that makes it nearly impossible to find two different inputs that result in the same hash value.
    

### Command Line Interface (CLI)

- **Definition:** A character-driven interface for accessing computers without graphical icons.
    

### Compliance

- **Definition:** Ensuring adherence to applicable laws, policies, or regulatory requirements.
    

### Confidentiality

- **Definition:** The security service of preventing access to resources by unauthorized users.
    

### Cross-Site Scripting (XSS)

- **Definition:** The malicious insertion of scripting code onto a vulnerable website.
    

### Cryptography

- **Definition:** The art and science of hiding information through encryption and decryption.
    

---

## D

### Data Leakage Prevention (DLP)

- **Definition:** Technology that uses deep analysis to ensure confidential information does not leave the network.
    

### Data Link Layer (Layer 2)

- **Definition:** The OSI layer responsible for physical addressing (MAC addresses) and network topology.
    

### Data Sovereignty

- **Definition:** The concept that data is subject to the laws of the country where it is physically stored.
    

### Decryption

- **Definition:** The process of converting ciphertext back into usable plaintext.
    

### Default Deny

- **Definition:** A security stance that blocks all access until a valid, explicit exception is defined.
    

### Defense in Depth

- **Definition:** Using multiple layers of security components to create a cumulative defensive effect.
    

### Demilitarized Zone (DMZ)

- **Definition:** A perimeter network segment used to host public-facing resources (like web servers) separate from the internal network.
    

### Digital Signature

- **Definition:** A mechanism for proving the source and integrity of a dataset using public key cryptography.
    

### Disaster Recovery Plan (DRP)

- **Definition:** A plan to restore mission-critical functions after they have been interrupted by an adverse event.
    

### DNS Poisoning / Spoofing

- **Definition:** Falsifying DNS server data so that queries return incorrect IP addresses, often used for man-in-the-middle attacks.
    

---

## E

### Eavesdropping

- **Definition:** Listening in on digital conversations, usually requiring a sniffer or protocol analyzer.
    

### Egress Filtering

- **Definition:** Filtering traffic as it attempts to leave a network.
    

### Encapsulation

- **Definition:** Enclosing one protocol or packet inside another (tunneling).
    

### Encryption

- **Definition:** Converting original data into an unusable form to protect it from unauthorized parties.
    

### Endpoint Detection and Response (EDR)

- **Definition:** Monitoring endpoint devices from a localized dashboard to identify and respond to threats.
    

---

## F

### Fail-Closed / Fail-Open

- **Fail-Closed:** A failure results in no access (most secure).
    
- **Fail-Open:** A failure results in unrestricted access.
    

### False Negative / False Positive

- **False Negative:** A malicious event that does _not_ trigger an alert.
    
- **False Positive:** A benign event that _does_ trigger a false alarm.
    

---

## G

### Gateway

- **Definition:** An entrance or exit point to a controlled network space.
    

### Governance, Risk, and Compliance (GRC)

- **Definition:** Applying security technologies in alignment with legal and regulatory requirements.
    

---

## H

### Hardening

- **Definition:** Securing a host by removing unnecessary software, patching, and imposing secure configurations.
    

### Hashing

- **Definition:** Using mathematical rules to produce a unique "fingerprint" (hash value) for a dataset to verify integrity.
    

### Honeypot

- **Definition:** A trap system designed to distract hackers from real targets and learn their methods.
    

---

## I

### Incident Response

- **Definition:** The planned reaction to negative security situations or events.
    

### Ingress Filtering

- **Definition:** Filtering traffic as it attempts to enter a network.
    

### Intrusion Detection System (IDS)

- **Definition:** A mechanism to detect unauthorized activities and attacks; can be passive (alerts) or active (disconnecting sessions).
    

### IP Security (IPSec)

- **Definition:** A suite of protocols used to secure IP communications by authenticating and encrypting each IP packet.
    

---

## L

### Least Privilege

- **Definition:** Providing users with only the minimum level of access necessary to perform their tasks.
    

### Load Balancing

- **Definition:** Spreading workload across multiple devices to maintain high performance and availability.
    

---

## M

### MAC Address (Media Access Control)

- **Definition:** The unique 48-bit physical hardware address assigned to a network interface.
    

### Man-in-the-Middle (MitM) Attack

- **Definition:** An attack where a hacker positions themselves between a client and server to eavesdrop or manipulate communications.
    

---

## N

### Network Address Translation (NAT)

- **Definition:** A service that converts internal private IP addresses to external public addresses to hide internal network details.
    

### Next-Generation Firewall (NGFW)

- **Definition:** A firewall that goes beyond traditional packet filtering to include application-level inspection and intrusion prevention.
    

---

## P

### Packet / Frame

- **Packet:** Data at the Network Layer (Layer 3).
    
- **Frame:** Data at the Data Link Layer (Layer 2).
    

### Phishing

- **Definition:** Luring victims to an attack site or tricking them into revealing information via false credentials (often email).
    

### Public Key Infrastructure (PKI)

- **Definition:** A framework that uses digital certificates and cryptography to provide secure identification and communication.
    

---

## R

### Risk Assessment

- **Definition:** Examining asset values, threat levels, and the cost of compromise versus the cost of protection.
    

### Rootkit

- **Definition:** Malware that replaces OS components with altered code to maintain hidden access to a system.
    

---

## S

### SIEM (Security Information and Event Management)

- **Definition:** A toolset for collecting and aggregating relevant info from multiple systems for security decision-making.
    

### Social Engineering

- **Definition:** Manipulating people into performing tasks or releasing information that violates security.
    

### Stateful Inspection

- **Definition:** Tracking the state of active connections to allow inbound responses to previous outbound requests.
    

---

## T

### Transport Layer Security (TLS)

- **Definition:** The successor to SSL; a protocol that provides secure communication over a computer network.
    

### Trojan Horse

- **Definition:** Malicious code disguised as a benign or useful program.
    

---

## V

### Virtual Private Network (VPN)

- **Definition:** A mechanism to establish a secure, encrypted connection across an insecure intermediary network (like the internet).
    

---

## Z

### Zero-Day Exploit

- **Definition:** An attack that exploits a previously unknown vulnerability for which no defense currently exists.
    

### Zombie

- **Definition:** A compromised computer that is part of a botnet.
